# 文件组织规范

> 保持项目整洁，每个文件都有它的归属

---

## 🎯 核心原则

```
┌─────────────────────────────────────────┐
│  禁止随意在根目录创建文件！              │
│                                         │
│  根目录只保留3个文件：                   │
│  - README.md    项目说明                │
│  - CLAUDE.md    项目配置                │
│  - CURRENT_TASK.md  当前任务            │
│                                         │
│  其他所有文件必须放入对应的目录！        │
└─────────────────────────────────────────┘
```

---

## 📂 目录归属对照表

### 重要：全局文档 vs 项目文档

在创建文档前，先判断是**全局文档**还是**项目文档**：

```
┌─────────────────────────────────────────┐
│  判断标准：                              │
│                                         │
│  🌐 全局文档（放 F:\claude-code\docs\）  │
│  ✅ Plugin/MCP 使用指南                 │
│  ✅ Claude Code 通用功能说明            │
│  ✅ 与具体项目无关的文档                │
│                                         │
│  📁 项目文档（放 {项目}\docs\）          │
│  ✅ 项目特定的功能指南                  │
│  ✅ 项目内部的模板和规范                │
│  ✅ 与项目强相关的文档                  │
└─────────────────────────────────────────┘
```

### 按文件类型查找

| 你想创建... | 放到哪个目录 |
|-------------|--------------|
| **Plugin/MCP文档** | **`F:\claude-code\docs/`** *(全局)* |
| **Claude Code通用文档** | **`F:\claude-code\docs/`** *(全局)* |
| 项目特定指南 | `{项目}\docs/guides/` |
| 项目模板 | `{项目}\docs/templates/` |
| 技能模块 | `{项目}\skills/{技能名}/` |
| 技能设计指南 | `{项目}\guides/` |
| 子项目 | `{项目}\projects/{新项目ID}/` |
| 参考资料 | `{项目}\references/` |
| 复盘总结 | `{项目}\retrospectives/` |
| 学习日志 | `{项目}\logs/` |
| **不确定放哪里** | **`{项目}\temp/inbox/`** |

## 🔄 文件生命周期

### 新建文件流程

```
1. 🤔 思考：这个文件属于什么类型？
   │
2. 🔍 查表：对照上面的"目录归属表"
   │
3. 📂 放置：放入对应目录
   │
   └─ 如果不确定 → 放入 temp/inbox/
```

### 临时文件处理流程

```
temp/inbox/ (收件箱)
     │
     ├─→ 确定归属 → 移动到正式目录
     │
     └─→ 需要编辑 → 移动到 temp/draft/

temp/draft/ (草稿)
     │
     ├─→ 完成编辑 → 移动到正式目录
     │
     └─→ 不再需要 → 移动到 temp/archive/

temp/archive/ (归档)
     │
     └─→ 确认无用 → 删除
```

## ✅ 创建文件检查清单

在创建任何新文件之前，问自己：

- [ ] **有现成的目录吗？** → 检查现有目录结构
- [ ] **是文档资源吗？** → 放入 `docs/` 对应子目录
- [ ] **是技能相关吗？** → 放入 `skills/` 或 `guides/`
- [ ] **是新项目吗？** → 放入 `projects/`
- [ ] **不确定归属？** → **放入 `temp/inbox/`**

---

## 🚨 常见错误

| 错误做法 | 正确做法 |
|----------|----------|
| ❌ 在根目录创建 `新功能.md` | ✅ 放入 `docs/guides/新功能.md` |
| ❌ 在根目录创建 `技能复盘.md` | ✅ 放入 `retrospectives/技能复盘.md` |
| ❌ 在根目录创建 `临时笔记.md` | ✅ 放入 `temp/inbox/临时笔记.md` |
| ❌ 随意创建新目录 | ✅ 先检查是否已有合适目录 |

```
┌─────────────────────────────────────────┐

│  - 整理或删除收件箱中的文件             │
│                                         │
│  每月检查一次 temp/draft/               │
│  - 完成草稿或归档                       │
│                                         │
│  每季检查一次 temp/archive/             │
│  - 删除确认无用的归档文件               │

```
