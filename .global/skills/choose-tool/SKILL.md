# 开发工具选择技能

## 目的

分析用户任务，自动推荐最合适的开发工具和工作流程。

## 可用工具

### 1. agent-skills (AI Agent 协作系统)
- **用途**：项目立项可行性分析
- **场景**：
  - 新项目立项
  - 产品需求分析
  - 文档整理和结构化
  - 从分析到玩法的完整流程
- **技能**：doc-organizer, business-analyst, devil-advocate, social-game-designer

### 2. superpowers (软件开发工作流程)
- **用途**：完整的软件开发项目
- **场景**：
  - 需要系统化流程的复杂任务
  - 强调 TDD 和代码质量
  - 从头脑风暴到实现的完整流程
- **技能**：brainstorming, writing-plans, executing-plans, test-driven-development

### 3. OpenSpec (规范管理)
- **用途**：重要变更的规范管理
- **场景**：
  - 重要变更需要规范
  - 架构调整需要审批
  - 需求模糊需要权威规范
- **触发**：自动检测触发条件

### 4. feature-dev (Plugin)
- **命令**：`/feature-dev`
- **用途**：快速实现单个功能
- **场景**：
  - 中小型开发任务
  - 理解现有代码库
  - 快速功能迭代

### 5. frontend-design (Plugin)
- **命令**：`/frontend-design`
- **用途**：前端界面设计
- **场景**：
  - UI/UX 设计
  - 前端组件开发
  - 界面代码生成

### 6. code-review (Plugin)
- **命令**：`/code-review`
- **用途**：代码审查
- **场景**：
  - PR 代码审查
  - 代码质量检查
  - 改进建议提供

## 决策流程

### 第一步：分析任务类型

向用户提问以确定任务类型：

```
请选择任务类型：

1. 📊 项目立项 - 新项目分析、需求分析、可行性研究
2. 📐 规范提案 - 重要变更、架构调整、需要审批的提案
3. 💻 功能开发 - 添加新功能、修改现有功能、代码实现
4. 🎨 前端设计 - UI/UX设计、界面开发、组件设计
5. 🔍 代码审查 - PR审查、代码质量检查、改进建议

请回复数字或描述您的任务。
```

### 第二步：根据选择推荐工具

#### 选择 1：项目立项 → agent-skills

```
推荐工具：agent-skills

流程：
1. doc-organizer - 文档整理
2. business-analyst - 商业分析
3. devil-advocate - 批判审查（可选）
4. social-game-designer - 玩法设计

是否开始？
```

#### 选择 2：规范提案 → OpenSpec

```
推荐工具：OpenSpec

流程：
1. 检查触发条件
2. 创建规范提案
3. 等待审批
4. 审批后实施

是否继续？
```

#### 选择 3：功能开发

```
请选择复杂度：

A. 简单功能 - 单个功能、快速实现
   → 推荐工具：feature-dev
   → 命令：/feature-dev

B. 完整项目 - 需要系统化流程
   → 推荐工具：superpowers
   → 自动技能调用

请回复 A 或 B。
```

#### 选择 4：前端设计 → frontend-design

```
推荐工具：frontend-design
命令：/frontend-design

是否使用此工具？
```

#### 选择 5：代码审查 → code-review

```
推荐工具：code-review
命令：/code-review

是否使用此工具？
```

## 自动检测规则

如果用户任务描述明确，可以直接推荐工具，无需询问：

### 检测关键词

| 关键词 | 推荐工具 |
|--------|----------|
| "立项"、"可行性分析"、"项目分析" | agent-skills |
| "提案"、"规范"、"架构变更" | OpenSpec |
| "添加功能"、"实现功能" | feature-dev 或 superpowers |
| "设计页面"、"UI设计"、"前端" | frontend-design |
| "审查代码"、"PR"、"代码质量" | code-review |
| "重构"、"优化"（大规模） | superpowers + OpenSpec |
| "社交游戏"、"玩法" | agent-skills |

---

## 快速决策

### 简单功能 vs 完整项目

```
判断标准：
- 影响 < 3 个文件 → feature-dev
- 影响 > 5 个文件 → superpowers
- 需要新架构 → superpowers + OpenSpec
- 涉及多个团队 → OpenSpec
```

## 示例对话

### 示例 1：明确任务

**用户**："我要做一个社交游戏项目，需要做可行性分析"

**AI**：
```
🎯 推荐工具：agent-skills

📋 任务分析：项目立项可行性分析

✅ 推荐理由：这是一个新项目，需要完整的可行性分析流程

🔄 流程：
  1. doc-organizer - 整理项目文档
  2. business-analyst - 商业和产品分析
  3. devil-advocate - 批判性审查（可选）
  4. social-game-designer - 玩法设计

🚀 开始使用：
  是否开始 agent-skills 流程？
```

### 示例 2：需要选择

**用户**："我要开发一个新功能"

**AI**：
```
请选择复杂度：

A. 简单功能 - 单个功能、快速实现
   → 使用：/feature-dev

B. 完整项目 - 需要系统化流程
   → 使用：superpowers

请回复 A 或 B，或描述更多细节。
```
