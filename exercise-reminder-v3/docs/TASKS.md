## 🔧 使用说明

### 添加新任务

```markdown
### [ ] P[0-2] 任务名称
**创建时间**：YYYY-MM-DD
**描述**：[任务描述]
**依赖**：[其他任务或资源]
```

### 更新任务状态

1. 移动任务到对应状态区域
2. 更新进度和子任务
3. 在状态变更日志中记录

### 任务完成

1. 移动任务到 "已完成"
2. 更新 `docs/WORK-LOG.md`
3. 从本文件中删除（或保留最近 5 条）

## 📊 优先级定义

| 优先级 | 说明 | 示例 |
|--------|------|------|
| **P0** | 最高优先级，阻塞其他任务 | 核心功能、阻塞性 bug |
| **P1** | 高优先级，重要但不紧急 | 功能增强、性能优化 |
| **P2** | 中等优先级，可以延后 | 文档完善、代码重构 |
