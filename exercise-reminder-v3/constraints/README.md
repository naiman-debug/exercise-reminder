## 📋 约束文档清单

### 必须遵守的约束

所有 AI Agent 在开发此项目时，必须遵守以下约束文档：

#### 1. 工作流恢复约束 ⭐ **最重要**

**文档**：[workflow-recovery.md](./workflow-recovery.md)

**作用**：
- 确保新会话能够快速恢复工作状态
- 避免重复工作，保持开发连续性

**强制规则**：
- ✅ 每次会话启动必须先读取 `docs/WORK-LOG.md` 最后 20 行
- ✅ 自动汇报：上次完成什么、下一步是什么、用了什么工具
- ✅ 识别快速恢复命令："继续"/"恢复"/"continue"/"resume"/"next"/"下一步"/"接着来"
- ✅ 完成任何模块后立即更新 WORK-LOG.md
- ✅ 更新必须包含：总体进度%、待开发清单

**触发命令**：
```
继续 / 恢复 / continue / resume / next / 下一步 / 接着来
```

**恢复响应格式**：
```
[工作状态恢复]
📌 最后完成：[模块名]
📋 下一步：[待开发首项]
🛠️ 工具：[MCP/Skills]
📊 当前进度：[X]%
是否继续？
```

## 🎯 使用流程

### 新会话启动时

```
1. 读取 constraints/workflow-recovery.md
2. 读取 docs/WORK-LOG.md 最后 20 行
3. 按照恢复格式汇报当前状态
4. 等待用户确认
```

### 完成任务后

```
1. 更新 docs/WORK-LOG.md
2. 记录：
   - 完成的任务和文件
   - 使用的技术和方法
   - 下一步计划
   - 当前进度总览
```
